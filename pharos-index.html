⚙️ x402 Payment Flow on Pharos
1
Agent sends POST /skill/audit
AI Agent calls GuardianSkill with a contract address. No pre-registration, no API key, no subscription needed.
Pharos Agent → GuardianSkill Server
2
Server returns HTTP 402 + payment instructions
GuardianSkill responds with payment requirements: 0.01 USDC on Pharos ChainID 688689, recipient address, x402 scheme.
x402 Protocol · USDC · eip155:688689
3
Agent's x402 client auto-pays on Pharos
The x402 SDK automatically signs and sends 0.01 USDC on Pharos. Sub-second finality — confirmed in <1 second at near-zero gas cost.
Pharos: 30,000 TPS · Sub-second finality
4
GuardianSkill runs the audit pipeline
Fetches bytecode from Pharos RPC → runs bytecode heuristics (SELFDESTRUCT, DELEGATECALL, tx.origin) → optionally runs LLM semantic analysis for source code.
viem · Pharos RPC · LLM (OpenAI-compatible)
5
Structured JSON response delivered (HTTP 200)
Agent receives: risk_score, risk_level, safe_to_transact flag, findings list, recommendation. PAYMENT-RESPONSE header contains tx_hash proof.
safe_to_transact → Agent gates next action
🔧 Agent Integration Code
Any Pharos Agent integrates GuardianSkill in 3 lines:
// 1. Wrap fetch with x402 auto-payment const fetchWithPayment = wrapFetchWithPayment(fetch, x402Client); // 2. Call GuardianSkill — auto-pays 0.01 USDC on Pharos if 402 returned const audit = await fetchWithPayment("http://skill-server/skill/audit", { method: "POST", body: JSON.stringify({ contract_address: "0x...", chain: "pharos" }) }); // 3. Gate transaction on audit result if (audit.safe_to_transact) { executeTransaction(); // proceed } else { alertUser(audit.recommendation); // abort }
